{/* # Job Management */}

Monitor and manage the status of your document processing jobs.

## List Jobs

<ParamField path="/parse/jobs" method="GET">
  List all processing jobs with pagination
</ParamField>

### Query Parameters

<ResponseField name="page" type="number" default="1">
  Page number for pagination
</ResponseField>

<ResponseField name="page_size" type="number" default="20">
  Number of items per page (max: 100)
</ResponseField>

<ResponseField name="status" type="string">
  Filter by job status (pending/processing/completed/failed)
</ResponseField>

<RequestExample>
```bash cURL
curl "https://api.invaro.ai/api/v1/parse/jobs?page=1&page_size=20&status=completed" \
  -H "Authorization: Bearer your_api_key"
```

```python Python
import requests

url = "https://api.invaro.ai/api/v1/parse/jobs"
headers = {
    "Authorization": "Bearer your_api_key"
}
params = {
    "page": 1,
    "page_size": 20,
    "status": "completed"
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  'https://api.invaro.ai/api/v1/parse/jobs?page=1&page_size=20&status=completed',
  {
    headers: {
      'Authorization': 'Bearer your_api_key'
    }
  }
);

const data = await response.json();
console.log(data);
```
</RequestExample>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Contains the list of jobs and pagination info
  <Expandable title="properties">
    <ResponseField name="jobs" type="array">
      Array of job objects
      <Expandable title="properties">
        <ResponseField name="job_id" type="string">
          Unique identifier for the job
        </ResponseField>
        <ResponseField name="status" type="string">
          Current status of the job
        </ResponseField>
        <ResponseField name="created_at" type="string">
          Job creation timestamp
        </ResponseField>
        <ResponseField name="completed_at" type="string">
          Job completion timestamp (if completed)
        </ResponseField>
        <ResponseField name="file_count" type="number">
          Number of files in the job
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="pagination" type="object">
      Pagination details
      <Expandable title="properties">
        <ResponseField name="total" type="number">
          Total number of jobs
        </ResponseField>
        <ResponseField name="page" type="number">
          Current page number
        </ResponseField>
        <ResponseField name="page_size" type="number">
          Items per page
        </ResponseField>
        <ResponseField name="total_pages" type="number">
          Total number of pages
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Cancel Job

<ParamField path="/parse/jobs/{job_id}" method="DELETE">
  Cancel a processing job
</ParamField>

<RequestExample>
```bash cURL
curl -X DELETE "https://api.invaro.ai/api/v1/parse/jobs/job_123abc" \
  -H "Authorization: Bearer your_api_key"
```

```python Python
import requests

url = "https://api.invaro.ai/api/v1/parse/jobs/job_123abc"
headers = {
    "Authorization": "Bearer your_api_key"
}

response = requests.delete(url, headers=headers)
print(response.json())
```

```javascript JavaScript
const response = await fetch('https://api.invaro.ai/api/v1/parse/jobs/job_123abc', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer your_api_key'
  }
});

const data = await response.json();
console.log(data);
```
</RequestExample>

### Response Example

```json
{
  "success": true,
  "data": {
    "job_id": "job_123abc",
    "status": "cancelled",
    "cancelled_at": "2024-01-07T10:02:00Z"
  }
}
```

## Error Responses

### Job Not Found
```json
{
  "success": false,
  "error": {
    "code": "JOB_NOT_FOUND",
    "message": "Job not found"
  }
}
```

### Invalid Job ID
```json
{
  "success": false,
  "error": {
    "code": "INVALID_JOB_ID",
    "message": "Invalid job ID format"
  }
}
```

### Cannot Cancel Job
```json
{
  "success": false,
  "error": {
    "code": "CANNOT_CANCEL_JOB",
    "message": "Cannot cancel a completed job"
  }
}
```

## Best Practices

1. **Job Monitoring**
   - Regularly check job status for long-running jobs
   - Implement exponential backoff for status checks
   - Store job IDs for future reference

2. **Error Handling**
   - Handle all possible job statuses
   - Implement retry logic for failed jobs
   - Log job details for debugging

3. **Resource Management**
   - Cancel unnecessary jobs to free up resources
   - Monitor job completion times
   - Use appropriate page sizes when listing jobs
