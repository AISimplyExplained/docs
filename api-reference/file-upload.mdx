# File Upload

The file upload endpoint allows you to submit documents for processing. This is typically the first step in using the Invaro API for document parsing.

## Upload Endpoint

<ParamField path="/parse/upload" method="POST">
  Upload one or more files for processing
</ParamField>

### Request Format

The request should use `multipart/form-data` encoding to upload files.

<RequestExample>
```bash
curl -X POST "https://api.invaro.ai/api/v1/parse/upload" \
  -H "Authorization: Bearer your_api_key" \
  -F "files=@statement.pdf" \
  -F "files=@invoice.jpg"
```
</RequestExample>

### File Requirements

<Card>
  - Supported formats: PDF, JPG, JPEG, PNG
  - Maximum file size: 10MB per file
  - Maximum files per request: 50
  - Minimum resolution for images: 300 DPI recommended
</Card>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the upload was successful
</ResponseField>

<ResponseField name="data" type="object">
  Contains the upload results
  <Expandable title="properties">
    <ResponseField name="files" type="array">
      Array of processed files
      <Expandable title="properties">
        <ResponseField name="file_id" type="string">
          Unique identifier for the uploaded file
        </ResponseField>
        <ResponseField name="file_url" type="string">
          Temporary URL to access the file
        </ResponseField>
        <ResponseField name="error" type="string">
          Error message if file processing failed, null otherwise
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="status" type="string">
      Overall status of the upload operation
    </ResponseField>
  </Expandable>
</ResponseField>

### Success Response Example

```json
{
  "success": true,
  "data": {
    "files": [
      {
        "file_id": "1234567890_statement.pdf",
        "file_url": "https://storage.invaro.ai/files/1234567890_statement.pdf",
        "error": null
      },
      {
        "file_id": "1234567890_invoice.jpg",
        "file_url": "https://storage.invaro.ai/files/1234567890_invoice.jpg",
        "error": null
      }
    ],
    "status": "done"
  }
}
```

### Error Response Example

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid file type",
    "details": "Only PDF, JPG, JPEG, and PNG files are allowed"
  }
}
```

## Best Practices

1. **File Preparation**
   - Ensure files are not corrupted
   - Compress images when possible while maintaining quality
   - Remove password protection from PDFs
   - Ensure documents are properly oriented

2. **Efficient Uploading**
   - Use batch uploads when possible
   - Implement retry logic for failed uploads
   - Verify file integrity using checksums
   - Monitor upload progress for large files

3. **Error Handling**
   - Always check the success field in responses
   - Handle partial success cases where some files fail
   - Implement appropriate timeout handling
