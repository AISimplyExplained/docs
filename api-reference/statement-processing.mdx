{/* # Bank Statement Processing */}

Extract structured data from bank statements using our advanced OCR and AI processing capabilities.

## Process Statements Endpoint

<ParamField path="/parse/statements" method="POST">
  Process uploaded bank statements for data extraction
</ParamField>

### Request Format

<ResponseField name="file_ids" type="array" required>
  Array of file IDs obtained from the upload endpoint
</ResponseField>

<RequestExample>
```bash
curl -X POST "https://api.invaro.ai/api/v1/parse/statements" \
  -H "Authorization: Bearer your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "file_ids": ["1234567890_statement.pdf"]
  }'
```
</RequestExample>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the processing request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Contains the processing job details
  <Expandable title="properties">
    <ResponseField name="job_id" type="string">
      Unique identifier for the processing job
    </ResponseField>
    <ResponseField name="status" type="string">
      Current status of the job (processing/completed)
    </ResponseField>
    <ResponseField name="estimated_completion" type="string">
      Estimated completion time in ISO 8601 format
    </ResponseField>
  </Expandable>
</ResponseField>

### Success Response Example

```json
{
  "success": true,
  "data": {
    "job_id": "job_123abc",
    "status": "processing",
    "estimated_completion": "2024-01-07T10:05:00Z"
  }
}
```

## Best Practices

1. **File Preparation**
   - Ensure statements are in PDF format
   - Remove password protection
   - Use high-quality scans (300 DPI minimum)
   - Ensure all pages are properly oriented

2. **Processing Strategy**
   - Process statements in chronological order
   - Use batch processing for multiple statements
   - Implement webhook notifications for long-running jobs

3. **Error Handling**
   - Validate file IDs before processing
   - Implement retry logic for failed jobs
   - Store job IDs for future reference

## Error Responses

### Invalid File ID
```json
{
  "success": false,
  "error": {
    "code": "INVALID_FILE_ID",
    "message": "One or more file IDs are invalid or expired"
  }
}
```

### Processing Failed
```json
{
  "success": false,
  "error": {
    "code": "PROCESSING_FAILED",
    "message": "Failed to process statement",
    "details": "Unable to detect bank template"
  }
}
```
